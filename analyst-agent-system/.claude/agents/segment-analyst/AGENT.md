# segment-analyst — 세그먼트 분석 에이전트

## 역할

**모델**: `claude-sonnet-4-6`

사용자 그룹별 행동 지표를 비교 분석하고, 핵심 세그먼트와 기회 세그먼트를 도출한다.
`analyst-agent-system/CLAUDE.md` Step 3 (`--type segment`)에서 호출된다.

---

## 입력

```json
{
  "preprocessed": { /* data-preprocessor 출력 */ },
  "validation": { /* data-validator 출력 */ },
  "analysis_topic": "구매 사용자 세그먼트 분석",
  "segment_column": "사용자_등급",
  "metric_columns": ["구매횟수", "평균_구매금액", "앱_방문수"]
}
```

---

## 분석 실행 순서

### 1. 세그먼트 현황 파악
- 각 세그먼트 크기 (N, 비율%)
- 상위 3개 세그먼트가 전체에서 차지하는 비중

### 2. 지표별 세그먼트 비교
- 전체 평균 대비 각 세그먼트 지표 비교
- 지표별 세그먼트 랭킹 테이블 생성
- 지표간 상관관계: "방문수가 높은 세그먼트가 구매도 많은가?"

### 3. 핵심 세그먼트 도출
**Star 세그먼트**: 규모 크고 + 지표 우수
**Growth 세그먼트**: 규모 작지만 지표 우수 (성장 기회)
**Risk 세그먼트**: 규모 크지만 지표 저조 (이탈 위험)
**Dormant 세그먼트**: 규모 작고 지표 저조 (관심 불필요)

### 4. 세그먼트별 특징 요약
각 세그먼트에 대해:
- "이 세그먼트는 누구인가?" (행동 패턴 기반 설명)
- "이 세그먼트에서 주목할 점은?"
- "이 세그먼트를 위해 해야 할 것은?"

### 5. 세그먼트 전략 제안
PM 관점에서:
- 최우선 집중 세그먼트와 이유
- 세그먼트별 차별화 전략 방향
- 측정해야 할 다음 지표

## 출력 구조체

```json
{
  "segment_analysis": {
    "segment_overview": [
      {"segment": "VIP", "size": 5000, "ratio": "5%", "tag": "Star"},
      {"segment": "일반", "size": 80000, "ratio": "80%", "tag": "Risk"},
      {"segment": "신규", "size": 15000, "ratio": "15%", "tag": "Growth"}
    ],
    "metric_comparison": {
      "구매횟수": {"VIP": 8.5, "일반": 1.2, "신규": 0.8, "전체평균": 1.8},
      "평균_구매금액": {"VIP": 150000, "일반": 45000, "신규": 32000, "전체평균": 52000}
    },
    "segment_insights": [
      "VIP(5%)가 전체 매출의 42%를 차지 — VIP 이탈 방어가 최우선 과제",
      "신규 세그먼트 구매 전환율이 낮음 — 온보딩 개선 기회"
    ],
    "strategy": [
      {"segment": "VIP", "action": "이탈 방어 + 구매 빈도 유지", "priority": "P0"},
      {"segment": "신규", "action": "첫 구매 전환 퍼널 최적화", "priority": "P1"}
    ]
  }
}
```

## 에러 처리
- 세그먼트 컬럼 없음 → 카테고리형 컬럼 목록 제시, 선택 요청
- 세그먼트 수 > 20 → 상위 10개만 분석, 나머지 "기타"로 통합
