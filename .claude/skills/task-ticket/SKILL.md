# /task-ticket — MEMB Task 티켓 생성 에이전트

## 모델

`claude-haiku-4-5-20251001`

## 개요

대화 내용이나 작업 설명을 바탕으로 MEMB 프로젝트에 Jira Task 티켓을 제안하고 생성한다.
제목·본문·기한·우선순위를 자동 제안하며, 사용자 컨펌 후 생성한다.

---

## 사용법

```
/task-ticket [대화 내용 또는 작업 설명]
```

args가 없으면 "어떤 작업을 티켓으로 만들까요? 대화 내용이나 작업 설명을 붙여넣어주세요." 라고 묻는다.

---

## 예시

```
/task-ticket 오늘 팀장이랑 얘기했는데, CRM 세그먼트 필터에 구매 횟수 조건 추가해야 할 것 같아

/task-ticket
[슬랙 대화 내용]
팀장: 이번 주 안으로 개인화 추천 API 스펙 정리해줄 수 있어?
나: 네, 금요일까지 할게요

/task-ticket 아래 3개 작업 각각 티켓으로 만들어줘
1. 세그먼트 미리보기 기능 QA
2. 리텐션 캠페인 발송 조건 문서화
3. Amplitude 대시보드 공유
```

---

## 실행 규칙

1. `.claude/agents/task-ticket-creator/AGENT.md`를 읽어 티켓 제안 및 생성 규칙을 파악한다.

2. args를 파싱하여 작업 내용을 분석한다.
   - args가 없으면 입력 요청
   - 대화 내용이 포함된 경우 핵심 작업만 추출
   - 복수 작업이면 각각 별도 티켓으로 분리

3. AGENT.md의 **Step 2** 형식으로 티켓 초안을 제안한다.
   - 제목: 동사+목적어, 50자 이내
   - 본문: 배경 / 작업 내용 / 완료 기준(AC)
   - 메타데이터 표: Priority, Due Date, Labels, Assignee

4. **반드시 사용자 컨펌을 기다린다.** 컨펌 전에는 티켓을 생성하지 않는다.

5. 컨펌 후 AGENT.md의 **Step 4**대로 스크립트를 생성하고 실행한다.

6. 생성된 티켓 URL을 보고한다.

```

### 제목
{제목}

### 본문
**배경**
{배경}

**작업 내용**
- {항목}

**완료 기준 (AC)**
- [ ] {조건}

### 메타데이터
| Priority | Due Date | Labels | Assignee |
|----------|---------|--------|---------|
| {값} | {값} | {값} | {값} |

✅ 이대로 생성할까요? 수정할 항목이 있으면 말씀해주세요.
```

```

🔗 {Jira URL}/browse/MEMB-{번호}
📋 {제목}
📅 기한: {Due Date} | 🎯 {Priority}
```
