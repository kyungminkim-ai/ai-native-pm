# Phase 0: Pre-PRD 가정 검증

## [사전 단계] gstack 참조 항목 확인

Phase 0 질문 시작 전, gstack 프레임워크에서 이번 PRD에 적용할 항목을 확인한다.
(출처: [github.com/garrytan/gstack](https://github.com/garrytan/gstack) — Garry Tan, YC President)

### A. `/office-hours` 강제 질문 매핑

| gstack 질문 | 핵심 포인트 | Phase 0 대응 |
|------------|-----------|------------|
| Demand Reality | 없어지면 진짜 화낼 행동 증거 | → Q1 수요 증거 (포함) |
| Status Quo | 지금 어떻게 해결하나, 그 비용은? | → Q2 현재 대안 (포함) |
| Desperate Specificity | 가장 필요한 실제 인간을 구체적으로 | → Q3 타깃 사용자 (포함) |
| Narrowest Wedge | 이번 주 돈 낼 최소 버전? | → Q4 최소 버전 (포함) |
| **Observation & Surprise** | 직접 보면서 예상과 달랐던 점 | → **Q6 (신규)** |
| **Future-Fit** | 3년 후 이 기능이 더 필수가 되는가? | → **Q7 (신규)** |

### B. `/plan-ceo-review` 스코프 모드 선택

이번 PRD의 방향성을 4가지 중 하나로 결정한다:

| 모드 | 설명 |
|------|------|
| **EXPANSION** | 10배 더 큰 버전 탐색 — 플랫폼화·확장 가능성 검토 |
| **SELECTIVE** | 기본 스코프 유지 + 확장 기회 병행 발굴 |
| **HOLD** | 기존 스코프 최대 rigorous — 실패 모드·엣지케이스 집중 (기본값) |
| **REDUCTION** | MVP만 남기고 나머지 제거 — 핵심 가치 1개로 축소 |

선택된 모드는 Phase 1 전략 방향 및 Phase 3 Self-Review 스코프 기준에 자동 반영.

## 실행 조건

아래 중 하나 해당 시 실행. 해당 없으면 Phase 1로 바로 진행.

| 조건 | 판단 기준 |
|------|---------|
| `--validate` 플래그 명시 | 사용자가 직접 요청 |
| Rough Note가 아이디어 수준 | "~하면 좋겠다", "~같은 게 있으면", 문장 3줄 이하 |
| 이니셔티브 ID 없이 새 주제 | TM-XXXX 없고 기존 PRD 파일도 없음 |

TM-XXXX가 있거나 Rough Note가 충분히 구체적이면 Phase 0 건너뜀.

> **Phase 0를 건너뛰는 경우에도** gstack 스코프 모드 확인은 Phase 1 시작 전에 수행한다.

## 7개 검증 질문 (하나씩 순서대로)

**Q1. 수요 증거** _(gstack: Demand Reality)_
> "이 기능이 없어지면 사용자가 불편해할 행동적 증거가 있나요?
> (예: 우회 방법 사용, 반복 요청, 비슷한 기능 사용률 등 — 관심 표현이 아닌 행동 기반으로)"

**Q2. 현재 대안** _(gstack: Status Quo)_
> "지금 사용자들이 이 문제를 어떻게 해결하고 있나요?
> 그 방법의 불편함이나 비용은 무엇인가요?"

**Q3. 타깃 사용자 구체화** _(gstack: Desperate Specificity)_
> "이 기능을 가장 필요로 하는 사용자를 구체적으로 묘사해주세요.
> (예: '앱푸시를 주 3회 이상 발송하는 마케터 중 타겟 설정에 30분 이상 쓰는 사람')"

**Q4. 최소 버전** _(gstack: Narrowest Wedge)_
> "지금 당장 출시할 수 있는 가장 좁은 버전은 무엇인가요?
> 전체 기능의 20%만 만들어도 핵심 가치를 전달할 수 있는 범위는?"

**Q5. 핵심 전제 3가지**
> "이 기획이 성립하려면 반드시 맞아야 하는 가정 3가지를 말씀해주세요.
> (예: '사용자가 설정을 직접 하려 한다', '현재 API가 이걸 지원한다')"

**Q6. 관찰과 발견** _(gstack: Observation & Surprise — 적용 여부에 따라 실행)_
> "실제로 사용자가 이 문제를 해결하는 과정을 직접 보거나 인터뷰한 적 있나요?
> 직접 보면서 예상과 달랐던 점, 놀라웠던 점은 무엇인가요?"

**Q7. 미래 적합성** _(gstack: Future-Fit — 적용 여부에 따라 실행)_
> "3년 후 시장이나 사용자 행동이 지금과 의미 있게 달라진다면,
> 이 기능은 더 필수적이 되나요, 덜 필수적이 되나요? 그 이유는 무엇인가요?"

## 완료 기준

- **사전 단계**: gstack 스코프 모드 + 추가 질문(Q6/Q7) 적용 여부 확인 완료
- Q1-Q5 + 선택된 추가 질문(Q6/Q7)에 대한 답변 확보
- 답변이 모호하면 1회 추가 확인 후 진행
- 사용자가 "바로 써줘"라고 하면 Q5(전제)만 확인 후 진행 (HOLD 모드 자동 적용)
