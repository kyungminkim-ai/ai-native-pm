# 플랫폼/API PRD 지원 규칙

## 판별 기준

Rough Note 또는 의도 구조화 결과에서 아래 키워드 1개 이상 감지 시 플랫폼 PRD로 분류.

| 판별 키워드 |
|-----------|
| API, 엔드포인트, endpoint, REST, GraphQL |
| SDK, 웹훅, webhook, 어댑터, 인터페이스 설계 |
| 플랫폼, platform, 공유 인프라, 공통 모듈 |
| 개발자 경험, DX, 온보딩, 문서화 |
| 버저닝, versioning, 하위 호환, deprecation |
| 인증/권한, auth, 토큰, API Key |

---

## 플랫폼 PRD 추가 섹션

기능 PRD 워크플로우 (2-B)를 그대로 따르되, **섹션 (4) 상세 기획** 내에 아래 서브시스템을 추가한다.

### [P] 플랫폼 인터페이스 설계

> 참고: `platform_prd.md` — API Design 항목

| ID | 요구사항 | 수용 기준 |
|----|---------|---------|
| P-001 | **[엔드포인트 설계]** {리소스 구조, HTTP 메서드, 요청/응답 포맷} | **Given/When/Then** |
| P-002 | **[에러 처리 규약]** {에러 코드 체계, 응답 포맷, Retry 정책} | |
| P-003 | **[버저닝 정책]** {버전 관리 방식, 하위 호환 기간, Deprecation 공지 방법} | |

**작성 시 체크리스트 (`platform_prd.md` 기준):**
- [ ] 리소스 구조 및 엔드포인트 설계 (동사/명사 혼용 금지)
- [ ] 요청/응답 포맷 (JSON 스키마, 필수/선택 필드 구분)
- [ ] HTTP 메서드 및 상태 코드 매핑
- [ ] 에러 코드 체계 및 메시지 표준
- [ ] 버저닝 전략 (`/v1/`, 헤더 기반 등)
- [ ] Pagination, Filtering, Sorting 지원 여부

---

### [S] 보안 및 접근 제어

> 참고: `platform_prd.md` — Security and access 항목

| ID | 요구사항 | 수용 기준 |
|----|---------|---------|
| S-001 | **[인증 방식]** {API Key / OAuth 2.0 / JWT 등 인증 메커니즘} | |
| S-002 | **[권한 모델]** {읽기/쓰기/관리 권한 분리, 스코프 정의} | |
| S-003 | **[데이터 노출 범위]** {민감 데이터 마스킹, 응답 필드 필터링} | |

**작성 시 체크리스트:**
- [ ] 인증 방식 및 토큰 만료 정책
- [ ] 권한 레벨 정의 (read-only / read-write / admin)
- [ ] Rate Limiting 정책 (요청 수 제한, Throttling)
- [ ] 개인정보 포함 여부 및 데이터 마스킹 기준

### [R] 신뢰성 및 SLA

> 참고: `platform_prd.md` — Reliability and performance 항목

| 항목 | 요구사항 | 기준 |
|------|---------|------|
| 가용성 | {업타임 목표} | {예: 99.9% / 월} |
| 지연시간 (p95) | {응답 시간} | {예: 200ms 이하} |
| 처리량 | {TPS/RPS} | {예: 1,000 req/s 이상} |
| 장애 복구 | {RTO / RPO} | {예: RTO 5분, RPO 1분} |

## 플랫폼 PRD 시 7-4 작업 시 주의사항 추가 항목

```
선행 확인 필요:
- API Contract 확정 (BE/FE 인터페이스 스펙)
- 네트워크 연동 확인 (크로스 계정, EKS 인바운드 허용 등)
- DB 스키마 설계 요청 (신규 테이블 DDL)
- 인증 자격증명 발급 (API Key, OAuth Client ID 등)
- 외부 팀 의존성 목록 작성 및 사전 공유
```

## 플랫폼 PRD Self-Review 추가 체크항목

| 검토 항목 | 통과 기준 |
|----------|----------|
| API 설계 일관성 | 엔드포인트 네이밍, 응답 포맷이 기존 API와 일관되는가 |
| 버저닝 명시 | 하위 호환 기간 및 Deprecation 정책이 명시되었는가 |
| 인프라 선행 작업 | 7-4 주의사항에 선행 작업 목록이 있는가 |
| 외부 팀 의존성 | 의존 팀이 인지하고 있는가 (OQ로 확인) |
