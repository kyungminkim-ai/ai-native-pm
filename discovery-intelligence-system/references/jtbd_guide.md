# JTBD(Job-to-be-Done) 프레임워크 가이드

## 개요

JTBD는 사용자가 "제품을 구매/사용한다"가 아닌, **"어떤 목적을 달성하기 위해 제품을 '고용'한다"**는 관점으로
사용자 니즈를 분석하는 프레임워크. 기능이 아닌 사용자의 진짜 목적에 집중한다.

---

## 1. 3계층 JTBD 구조

### 기능적 Job (Functional Job)
사용자가 실제로 **완수해야 하는 과업**.

**작성 형식:**
```
사용자는 {상황}에서 {과업}을 완료하고 싶다.
```

**예시:**
- ❌ "사용자는 앱을 빠르게 사용하고 싶다." (기능 수준)
- ✅ "사용자는 점심시간 20분 안에 오늘 입을 코디를 결정하고 싶다."

### 감정적 Job (Emotional Job)
과업 수행 중 사용자가 **느끼고 싶거나 피하고 싶은 감정**.

**작성 형식:**
```
사용자는 {과업 수행 중} {긍정 감정}을 느끼고, {부정 감정}을 피하고 싶다.
```

**예시:**
- ✅ "사용자는 코디 선택 과정에서 '나만의 취향'이라는 확신을 느끼고,
     '남들과 똑같아 보일 것 같은' 불안감을 피하고 싶다."

### 사회적 Job (Social Job)
과업 수행 결과로 사용자가 **타인에게 어떤 모습으로 보이고 싶은가**.

**작성 형식:**
```
사용자는 {상황}에서 {타인/집단}에게 {이미지/인상}으로 보이고 싶다.
```

**예시:**
- ✅ "사용자는 직장 동료들에게 '패션에 관심 있고 센스 있는 사람'으로 보이고 싶다."

## 2. JTBD 우선순위 매트릭스

| 차원 | 설명 | 질문 |
|------|------|------|
| **중요도** | 사용자에게 얼마나 중요한 Job인가 | "이 Job이 충족되지 않으면 얼마나 불편한가?" |
| **충족도** | 현재 시장이 얼마나 잘 충족하고 있는가 | "지금 사용하는 솔루션으로 얼마나 만족하는가?" |

**우선 기회 공식:**
```
기회 점수 = 중요도 + (중요도 - 충족도)
→ 점수가 높을수록 '미충족 기회'
```

| 중요도↑ 충족도↑ | 경쟁 과열 영역 (진입 신중) |
|--------------|----------------------|
| 중요도↑ 충족도↓ | **핵심 기회 영역** (여기에 집중) |
| 중요도↓ 충족도↑ | 과잉 충족 영역 (과투자 위험) |
| 중요도↓ 충족도↓ | 비기회 영역 (우선순위 낮음) |

## 3. 인터뷰에서 JTBD 추출하는 방법

### 핵심 탐침 패턴

| 발언 유형 | 탐침 방법 | JTBD 신호 |
|---------|---------|----------|
| "그냥..." | "그냥이라고 하셨는데, 왜 그 방법을 택하셨나요?" | 동기 탐색 |
| "귀찮아서..." | "어떤 부분이 가장 귀찮으세요? 구체적으로?" | 기능적 Job 하위 과업 |
| "기분이 이상해서..." | "어떤 기분이 드세요? 상황을 조금 더 말해주세요." | 감정적 Job |
| "남들이 보면..." | "어떻게 보이고 싶으세요?" | 사회적 Job |

### 발언에서 Job 추출하기

원문 발언:
> "솔직히 중고 거래에서 제일 시간 낭비가 '이 옷이 내 스타일인지 모르겠어서 물어보다 거래 날아가는' 상황이에요."

추출된 JTBD:
```
기능적: 사용자는 거래 전에 옷이 자신의 스타일과 맞는지 빠르게 판단하고 싶다.
감정적: 판단 과정에서 '내 취향에 맞다'는 확신을 얻고, '시간을 낭비했다'는 좌절을 피하고 싶다.
사회적: (이 발언에서 직접 추출 불가 — 추가 탐침 필요)
```

## 4. JTBD 작성 시 흔한 실수

| 실수 | 나쁜 예 | 좋은 예 |
|------|--------|--------|
| 기능 수준으로 작성 | "앱을 빨리 쓰고 싶다" | "출근 준비 5분 안에 오늘 코디를 결정하고 싶다" |
| 솔루션을 Job으로 작성 | "스타일 추천 기능을 쓰고 싶다" | "매일 아침 나에게 어울리는 옷을 확신하고 싶다" |
| 추상적 감정 | "좋은 경험을 하고 싶다" | "선택을 후회하지 않을 것 같다는 안도감을 느끼고 싶다" |
| 기업 관점 | "구매 전환율을 높이고 싶다" | (Job은 항상 사용자 관점) |
