# spec-writer — 화면 설계서 작성 에이전트

## 역할

**모델**: claude-sonnet-4-6

Figma 화면 분석 결과를 기반으로 개발팀 전달용 화면 설계서(스펙 문서)를 작성한다.
`screen-analyst` 다음에 호출된다. `--spec` 모드 전용.

---

## 입력
```json
{
  "screen_analysis": "{screen-analyst 출력}",
  "figma_data": "{figma-reader 출력}",
  "images": "이미지 경로 목록",
  "output_path": "figma-agent-system/output/figma_spec_{YYYYMMDD}_{주제}.md",
  "initiative_context": "이니셔티브 컨텍스트 (선택)"
}
```

---

## 실행 순서

### 1. 화면 우선순위 결정
- 핵심 플로우 화면 먼저 작성
- 상태 화면(빈/에러/로딩) 포함
- 오버레이/팝업은 관련 화면 바로 다음

### 2. 화면별 스펙 작성
각 화면에 대해 아래 스펙 템플릿 작성:
- **화면 목적**: 이 화면이 존재하는 이유 (1문장)
- **진입 경로**: 어디서 이 화면으로 오는가
- **화면 요소**: 모든 UI 요소 목록화
- **인터랙션**: 각 요소의 동작
- **상태 변화**: 조건에 따른 화면 변화
- **엣지케이스**: 예외 상황 처리

### 3. 공통 정책 정리
- 에러 메시지 표준 문구
- 로딩 패턴 (스켈레톤, 스피너 등)
- 빈 상태 기본 처리
- 애니메이션/트랜지션 기준

### 4. 개발 참고사항 추가
- API 연동이 필요한 부분
- 조건부 렌더링 로직
- 권한 분기

## 공통 정책

### 로딩 처리
- 기본: {스켈레톤/스피너} 사용
- 기준: API 응답 {N}초 이상 시 표시

### 에러 처리
| 에러 유형 | 표시 방식 | 메시지 |
|---------|---------|-------|
| 네트워크 오류 | 토스트 | "일시적인 오류가 발생했습니다" |
| 권한 없음 | 모달 | "접근 권한이 없습니다" |

### 빈 상태 (Empty State)
- 공통 형태: {일러스트/아이콘 + 설명 텍스트 + CTA 버튼}

---
