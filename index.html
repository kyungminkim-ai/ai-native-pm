Workflow Patterns

복합 워크플로우

자주 반복되는 작업 패턴을 하나의 명령 체인으로 실행합니다.

Pattern A
기획 파이프라인
아이디어 → Jira 티켓까지 완전 자동화
/discovery {topic} 시장 분석 + 페르소나
↓
/prd TM-XXXX PRD 9섹션 자동 생성
↓
/red {PRD file} 가정 검증 + 반론 질문
↓
/epic {PRD} TM-XXXX Jira Epic 티켓 자동 등록
Pattern B
주간 성과 보고
Flash + 회의록 + 이메일 배포 통합
/pgm --full MATCH {URL} Flash + 회의록 + Jira 코멘트
↓
/report {flash.md} C레벨 보고서 품질 검토
↓
/mail {flash.md} --to {팀} HTML 이메일 자동 배포
Pattern C
런치 파이프라인
Discovery → GTM → Confluence 게시
/discovery {topic} 시장 분석 리포트
↓
/prd [rough notes] PRD 문서 생성
↓
/gtm {PRD file} GTM Brief 자동 생성
↓
confluence-writer Confluence 자동 게시
Pattern D
과제 정리
CSV/JQL → 클러스터링 → Jira 등록
/ticket-review [CSV/JQL] MATCH팀 분류 + 유사도 클러스터링
↓
/epic {clusters} Jira Epic 자동 등록
Pattern H · New
데이터 → 기획
Databricks 분석 결과 → PRD 자동 연결
/databricks --analyze {table} KPI 데이터 분석 + 인사이트
↓
/prd TM-XXXX 분석 결과 → PRD Rough Note로 활용
↓
/red {PRD} 가정 검증
Pattern I · New
Slack → 회의록
채널 대화 수집 → 회의록 자동 생성
/slack --today #match-pm 오늘 대화 수집 + Action Item 추출
↓
/meeting --input {slack.md} 회의록 작성 + Confluence 업로드
↓
/pgm --weekly {Confluence URL} Jira Initiative 코멘트 자동 게시
Pattern J · New
Figma → 기획
화면 분석 → 설계서 + PRD 자동 생성
/figma [URL] 화면 구조 파악
↓
/figma [URL] --spec 개발팀 전달용 설계서 → Confluence
↓
/figma [URL] --prd PRD 초안 자동 생성 → /red → /epic

Design Principles

운영 철학

PM Studio는 다섯 가지 원칙 위에 설계되었습니다.

✍️
Prompt-First Thinking
빈 문서가 아닌 에이전트 요청으로 시작. 구조는 AI가, 판단은 PM이.
⚡
Draft Fast, Judge Slow
AI가 빠르게 초안을 만들고, PM이 깊이 검토한다. 속도와 품질을 분리.
🏗️
Context as Assets
지식이 구조화된 파일로 축적. context.md, decisions.md, references.json이 다음 작업의 기반.
🤝
Agents as Teammates
각 에이전트는 명확한 역할, 산출물, 품질 기준을 가진 팀원.
🧭
Human-in-the-Loop
전략, 고객 인사이트, 이해관계자 관리는 PM의 영역. AI는 실행을 지원.
